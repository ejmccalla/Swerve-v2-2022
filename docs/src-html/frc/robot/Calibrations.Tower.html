031    /** 
032     * The average wheel diameter (best: calibrate with encoders, ok: use tape measure).
033     *
034     * <p>To measure the average wheel diameter using the encoders requires a straight wall and chunk of carpet. The
035     * procedure goes as follows:
036     *
037     * <p>1. Place the carpet up against the wall.
038     *
039     * <p>2. Place a side of the robot flush against a straight wall and on the carpet.
040     *
041     * <p>3. Mark the front of the robot on the wall.
042     *
043     * <p>4. Zero the encoders.
044     *
045     * <p>5. Manually push the robot forward (keep the robot in contact with the wall the whole time) until the encoders
046     * indicate exactly 10 complete rotations.
047     *
048     * <p>6. Mark the front of the robot on the wall.
049     *
050     * <p>7. Measure the distance between the 2 marks on the wall.
051     *
052     * <p>8. The average wheel diameter is the measured distance divided by 10*pi.
053     *
054     * <p>This should be done before each event minimum
055     */
056    public static final double WHEEL_DIAMETER_INCH = 4.0; // TODO: Calibrate with encoders

059    /**
060     * Distance between right and left wheels.
061     *
062     * <p>To measure the track width, use the following procedure:
063     * 
064     * <p>1. Use a tape measure and record the distance between each pair of left and right wheels.
065     * 
066     * <p>2. The track width is the average of the two measurements from the first step.
067     *
068     * <p>Any changes to the track width from step 2 likely indicates there is mechanical damage to the module. Also, if the
069     * difference between the left/right wheel pairs is large, this indicates an "out-of-square" frame.
070     *
071     * <p>This should be done before each event minimum
072     */
073    public static final double TRACK_WIDTH_INCH = 19.5;

093    /**
094     * The absolute encoder angle for homing the wheel (best: calibrate with encoders, ok: use a straight edge to align the
095     * wheels).
096     *
097     * <p>To measure the absolute encoder home angle using the encoders requires a straight wall and chunk of carpet. The
098     * procedure goes as follows:
099     * 
100     * <p>1. Place the carpet up against the wall.
101     *
102     * <p>2. Place a side of the robot flush against a straight wall and on the carpet.
103     *
104     * <p>3. Start recording the absolute encoder values.
105     *
106     * <p>4. Manually push the robot forward (keep the robot in contact with the wall the whole time) for ~10 feet.
107     *
108     * <p>5. Stop recording the absolute encoder values.
109     *
110     * <p>6. The homing angle is the average of the recorded values. Depending on the initial alignment of the wheels, the
111     * beginning portion of the recorded values should be removed from the averaging.
112     *
113     * <p>The ordering of the values below are : front left, front right, rear left, rear right.
114     *
115     * <p>This should be done before each event minimum
116     */
117    public static final double[] ZEROS_RAD = {1.27, 0.91, 0.27, 5.82};

131    /**
132     * The maximum angular velocity used for constraining trajectory profile of the turning controller.
133     *
134     * <p>Measuring the maximum angular velocity of the turning motor requires a chunk of carpet.The procedure goes as
135     * follows:
136     * 
137     * <p>1. Place robot on the carpet.
138     *
139     * <p>2. Start recording the relative encoder velocity values.
140     *
141     * <p>3. Send full positive voltage to the turning motor.
142     *
143     * <p>4. Stop recording.
144     *
145     * <p>5. The maximum angular velocity is the average of the recorded values. Depending on the acceleration, the
146     * beginning portion of the recorded values should be removed from the averaging.
147     * 
148     * <p>6. Repeat steps 2-5 using a full negative voltage.
149     * 
150     * <p>7. Take 90% of the minimum from step 5 (positive/negative voltage commands) and use this as the maximum angular
151     * velocity. 
152     *
153     * <p>The ordering of the values below are : front left, front right, rear left, rear right.
154     *
155     * <p>This should be measured before each event minimum
156     */
157    public static final double[] MAX_TURN_VELOCITY_RPS = 
158        {10 * Math.PI, 10 * Math.PI, 10 * Math.PI, 10 * Math.PI}; // TODO: Part of turning FF calibration

161    /**
162     * The maximum angular acceleration used for constraining trajectory profile of the turning controller.
163     *
164     * <p>Measuring the maximum angular acceleration of the turning motor requires a chunk of carpet. The procedure goes as
165     * follows:
166     * 
167     * <p>1. Place robot on the carpet.
168     *
169     * <p>2. Start recording the relative encoder velocity values.
170     *
171     * <p>3. Send full positive voltage to the turning motor.
172     *
173     * <p>4. Stop recording.
174     *
175     * <p>5. The accelerations are the delta velocity measurements. Only keep the accleration values starting with the
176     * initial max voltage and ending with the motor reaching cruise velocity. Take the average of these remaining data.
177     * 
178     * <p>6. Since the acceleration data are noisy and short, repeat steps 3-5 several times. The maximum angular
179     * acceleration is the average of all the step 5's. 
180     * 
181     * <p>7. Repeat steps 2-6 using a full negative voltage.
182     * 
183     * <p>8. Take 90% of the minimum from step 6 (positive/negative voltage commands) and use this as the maximum angular
184     * acceleration. 
185     *
186     * <p>The ordering of the values below are : front left, front right, rear left, rear right.
187     *
188     * <p>This should be tuned before each event minimum
189     */
190    public static final double[] MAX_TURN_ACCELERATION_RPSS = 
191        {10 * Math.PI, 10 * Math.PI, 10 * Math.PI, 10 * Math.PI}; // TODO: Part of turning FF calibration
